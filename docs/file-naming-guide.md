# File Naming Guide

## Purpose

This file explains simple file naming rules for keeping the repository clean and easy to understand.

## Naming Rules

- Use lowercase letters
- Use hyphens between words
- Keep names short but clear
- Avoid random names
- Avoid spaces in file names
- Use consistent names across folders

## Good Examples

- project-summary.md
- training-notes.md
- evaluation-notes.md
- setup-instructions.md
- sample-results.md

## Bad Examples

- final final file.md
- newdocument.md
- test123.md
- random-notes.md
- my file.md

## Notes

Clear file names make the repository easier to review and maintain.
